Summary:
Implemented persistent command idempotency and sequence generation using a SQLite-backed command ledger, replacing in-memory cache/seq state.

Files touched:
- orchestration-server/app/command_ledger.py
- orchestration-server/app/main.py
- orchestration-server/app/models.py
- orchestration-server/README.md
- docs/TICKETS.md

Behavior change:
- Added durable command ledger at `orchestration-server/data/orchestration.db`.
- `request_id` idempotency now persists across process restarts.
- Duplicate request replay returns:
  - `idempotent_replay=true`
  - `reason_code=DUPLICATE_REQUEST`
- Same `request_id` with different payload now returns:
  - HTTP `409`
  - `reason_code=REQUEST_ID_PAYLOAD_MISMATCH`
- Concurrent identical request guard returns:
  - HTTP `409`
  - `reason_code=REQUEST_IN_PROGRESS`
- Command sequence allocation now comes from persistent ledger (`command_seq`) and remains monotonic.

Contract impact: additive

Risks/assumptions:
